### Question 2: Process Synchronization — The Reader-Writer Variant (20 Points)
Consider a shared database variable accessed by two categories of threads: **Readers** (who can access concurrently) and **Writers** (who require exclusive access). The following proposed entrance implementation is provided:

```c
// Shared Variables
int reader_count = 0;
pthread_mutex_t mock_mutex = PTHREAD_MUTEX_INITIALIZER;
sem_t database_sem; // Initialized to 1

void* Reader(void* arg) {
    pthread_mutex_lock(&mock_mutex);
    reader_count++;
    if (reader_count == 1) {
        sem_wait(&database_sem);
    }
    pthread_mutex_unlock(&mock_mutex);

    // --- CRITICAL SECTION: Read Database ---
    
    pthread_mutex_lock(&mock_mutex);
    reader_count--;
    if (reader_count == 0) {
        sem_post(&database_sem);
    }
    pthread_mutex_unlock(&mock_mutex);
    return NULL;
}

void* Writer(void* arg) {
    sem_wait(&database_sem);
    // --- CRITICAL SECTION: Write Database ---
    sem_post(&database_sem);
    return NULL;
}
```

1. **Starvation Invariant (10 Points):** Does this specific implementation introduce a starvation hazard for Writers? Prove your answer by tracing an execution timeline.
2. **Synchronization Correction (10 Points):** Propose a modification using an additional semaphore or condition queue that prevents Writer starvation by blocking new incoming Readers once a Writer announces its arrival.

```c

#include <sys/wait.h>

    int val = 5;

    if (pid == 0) {
        val += 10;
        if (fork() == 0) {
            val += 20;

            wait(NULL);
            printf("Child-Parent: val = %d\n", val);

}
```
