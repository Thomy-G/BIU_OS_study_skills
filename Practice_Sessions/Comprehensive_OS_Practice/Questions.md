### Part 2: Processes
**Question: Process Lifecycles & POSIX Fork Trees (20 Points)**

Analyze the following POSIX C program:

```c
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>
#include <stdlib.h>

int main() {
    printf("Base\n");
    pid_t pid1 = fork();
    if (pid1 == 0) {
        printf("Alpha\n");
        pid_t pid2 = fork();
        if (pid2 > 0) {
            wait(NULL);
            printf("Beta\n");
            exit(0);
        }
        printf("Gamma\n");
    } else {
        wait(NULL);
        printf("Delta\n");
    }
    printf("Omega\n");
    return 0;
}
```

1. **Process Tree Diagram (10 Points):** Draw or describe the process tree topology generated by this code (showing PIDs or labels for parent/child relationships, and where each process executes).
2. **Terminal Output Sequence (10 Points):** Write the exact terminal output sequence. If multiple sequences are possible due to concurrency, explain the scheduling factors and write one valid sequence.

### Part 4: Scheduling
**Question: CPU Scheduling & Context Switches (20 Points)**

Three processes ($P_1, P_2, P_3$) arrive in the ready queue at physical time $t = 0$. Their CPU burst requirements are:
* $P_1$: 8 ms
* $P_2$: 4 ms
* $P_3$: 3 ms

Assume the system incurs a **context-switch overhead of 1 ms** whenever the CPU switches execution from one process to another. No context switch is charged at $t = 0$ or at the final completion.

1. **Gantt Chart & Waiting Math (15 Points):** Draw the Gantt charts and calculate the Average Waiting Time (AWT) and Average Turnaround Time (ATT) for:
   - **First-Come, First-Served (FCFS)** (execution order $P_1 \rightarrow P_2 \rightarrow P_3$).
   - **Round Robin (RR)** with a time quantum $q = 3\text{ ms}$.
2. **System Overhead Trade-offs (5 Points):** In a system where response time is critical (such as an interactive user terminal), explain how the selection of the time quantum $q$ affects system overhead and user experience.

### Part 5: Deadlocks
**Question: Banker's Matrix & Safety (20 Points)**

A system has 4 processes ($P_0, P_1, P_2, P_3$) and 3 resource classes ($A, B, C$). The current allocation and demand state is defined as:

$$\textbf{Allocation} = \begin{pmatrix} P_0 & 0 & 1 & 0 \\ P_1 & 2 & 0 & 0 \\ P_2 & 3 & 0 & 2 \\ P_3 & 2 & 1 & 1 \end{pmatrix}, \quad \textbf{Max} = \begin{pmatrix} P_0 & 0 & 3 & 0 \\ P_1 & 3 & 2 & 2 \\ P_2 & 9 & 0 & 2 \\ P_3 & 2 & 2 & 2 \end{pmatrix}$$

$$\textbf{Total Resources Vector} = [8, 5, 7]$$

1. **Need Matrix Calculation (5 Points):** Compute the **Need Matrix**.
2. **Safety State Evaluation (10 Points):** Determine if the system is currently in a **Safe State**. If yes, provide a valid safe execution sequence and trace the `Available` vector step-by-step. If no, prove why a deadlock can occur.
3. **Dynamic Request Tracking (5 Points):** If $P_1$ requests resource vector $\textbf{Request}_{P_1} = [1, 0, 1]$, can the OS grant this request immediately? Prove your answer using the Banker's safety checks.
